RTO Support: геймдизайн
modular/rto_support/__docs/RTO_SUPPORT_GAME_DESIGN.md

1. Цель роли

RTO Support делает RTO отдельной support-ролью с четырьмя разными ритмами игры:

  1. выбрать пакет;
  2. занять позицию;
  3. развернуть сектор, если пакет боевой;
  4. отработать своё окно эффективности;
  5. уйти в recovery и пережить откат.

Точные цифры вынесены в RTO_SUPPORT_BALANCE.md. Этот документ фиксирует игровые роли пакетов.

2. Базовые правила

  • Один пресет на жизнь моба.
  • Один активный сектор на оператора.
  • Поддержка вызывается только через RTO-бинокль.
  • Mortar, CAS, Heavy Strike требуют активный сектор.
  • Logistics сектора не использует.

3. Стартовые пресеты

Mortar

Роль пакета:

  • частая утилита;
  • длинное окно сектора для последовательной работы одиночными минами;
  • одиночные HE, дымовые и зажигательные мины;
  • очень длинная перезарядка сектора после окончания окна.

Параметры:

  • радиус сектора: 7
  • длительность сектора: 100 секунд
  • кулдаун сектора: 300 секунд

Способности:

  • HE mortar: 1 снаряд, scatter 4, shared 4, personal 8
  • Smoke mortar: 1 снаряд, scatter 3, shared 3, personal 5
  • Incendiary mortar: 1 снаряд, scatter 4, shared 6, personal 10

Особенность:

  • при развёртывании сектора используется illumination payload с фальшфейером;
  • сами mortar-вызовы у RTO не повторяют апстримовый barrage и работают как одиночные выстрелы;
  • пакет построен так, чтобы игрок успел отработать серию одиночных мин за длинное окно, после чего ушёл в тяжёлую recovery-фазу.

CAS

Роль пакета:

  • средний по темпу авиационный пакет;
  • точечное сопровождение штурма;
  • заметно сильнее mortar по точности, но без heavy-инерции.

Параметры:

  • радиус сектора: 5
  • длительность сектора: 200 секунд
  • кулдаун сектора: 500 секунд

Способности:

  • Gun run: scatter 3, shared 12, personal 16
  • Laser run: scatter 2, shared 14, personal 18
  • Rocket barrage: scatter 4, shared 16, personal 20

Heavy Strike

Роль пакета:

  • дорогой ударный пакет;
  • длинное, но контролируемое окно применения;
  • умеренно длинные интервалы между ударами и длинная recovery-фаза сектора.

Параметры:

  • радиус сектора: 4
  • длительность сектора: 300 секунд
  • кулдаун сектора: 800 секунд

Способности:

  • Missile strike: scatter 2, shared 16, personal 20
  • Napalm strike: scatter 3, shared 18, personal 22

Особенность:

  • пакет использует самое длинное окно и самую тяжёлую recovery-фазу среди боевых пресетов;
  • окно рассчитано на дорогую, редкую поддержку, а не на частое повторное включение.

Logistics

Роль пакета:

  • долгий стратегический utility support;
  • снабжение, мины и турели;
  • пакет "на потом", а не для постоянного спама инфраструктурой.

Параметры:

  • сектора нет

Способности:

  • Supply drop: scatter 1, shared 120, personal 600
  • Mine crate drop: scatter 1, shared 90, personal 420
  • Mini-sentry drop: scatter 1, shared 120, personal 540
  • Full sentry drop: scatter 1, shared 180, personal 780

Особенность:

  • выгрузка запрещена в закрытые тайлы;
  • Full sentry drop остаётся самой дорогой логистической кнопкой.
